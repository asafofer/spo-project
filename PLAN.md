# Kueez Data Analysis & Validation Plan

## Objective

Validate the startup's core hypothesis: **Dynamic optimization (Bidder Selection, Timeouts, Floors) drives revenue lift.**
Instead of just "finding fruit," we are testing if the _conditions for optimization_ exist in the data.

## Context

- **Data Source:** A **daily slice** of a custom export from Kueez's internal system.
- **Origin:** Ad data generated by **Prebid.js**.
- **Goal:** Validate the startup hypothesis, not just optimize this specific dataset.

## Phase 1: Hypothesis Validation (Current Data) ✅

We will run three structured experiments to prove/disprove the premises of the startup's value proposition.

### 1. Bidder Selection Validation ✅

**Hypothesis:** Poor quality demand exists and causes drag (latency/waste) that can be eliminated.

- **Test File:** `analysis/tests/01_spo/TEST_01_SPO.md`
- **Script:** `analysis/tests/01_spo/analyze-bidder-efficiency.ts`
- **Status:** COMPLETED
- **Result:** Hypothesis VALIDATED - Found zombie bidders (sharethrough, pubmatic) and high latency issues (sovrn 50%, appnexusast 42% timeout rates).

### 2. Floor Price Segmentation Validation ✅

**Hypothesis:** Static floors are inefficient because value varies significantly by segment.

- **Test File:** `analysis/tests/02_floors/TEST_02_FLOORS.md`
- **Script:** `analysis/tests/02_floors/analyze-floor-segments.ts`
- **Status:** COMPLETED
- **Result:** Hypothesis VALIDATED - 12.6x eCPM spread (US Mobile $1.64 vs Kyrgyzstan $0.13). Cannot recommend specific floor values without losing bid data.

### 3. Timeout Impact Validation ✅

**Hypothesis:** Latency (proxied by timeouts) hurts revenue, and current static timeouts are suboptimal.

- **Test File:** `analysis/tests/03_timeouts/TEST_03_TIMEOUTS.md`
- **Script:** `analysis/tests/03_timeouts/analyze-timeout-impact.ts`
- **Status:** COMPLETED
- **Result:** Hypothesis PARTIALLY VALIDATED - 8 major bidders timeout 30-50% of requests. Cannot determine optimal timeout values without latency data.

- **Deliverable:** `analysis/docs/MISSING_DATA.md`
